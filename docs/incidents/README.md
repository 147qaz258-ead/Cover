# 事故复盘目录

> 本目录用于记录生产环境事故的复盘与预防措施。

---

## 目的 Purpose

系统性沉淀事故经验，避免同类问题复发。

## 适用范围 Scope

- 生产环境服务中断
- 数据丢失或损坏
- 安全漏洞
- 严重性能问题

## 当前状态 Status

**Active** - 持续记录

---

## 文件命名规范

```
incident-YYYYMMDD-<kebab-topic>.md
```

示例：
- `incident-20251222-api-timeout.md`
- `incident-20251223-r2-upload-failure.md`

---

## 复盘模板

每个事故文档应包含：

```markdown
# 事故复盘: [事故标题]

## 1. 概要
- **发生时间**: YYYY-MM-DD HH:MM
- **持续时长**: X 小时 Y 分钟
- **影响范围**: 描述受影响的用户/功能
- **严重等级**: P0/P1/P2/P3

## 2. 时间线
| 时间 | 事件 |
|------|------|
| HH:MM | 事件 1 |
| HH:MM | 事件 2 |

## 3. 根因分析 (RCA)
- 直接原因
- 根本原因

## 4. 修复措施
- 临时措施
- 永久修复

## 5. 预防措施
- [ ] 措施 1
- [ ] 措施 2

## 6. 经验教训
- 教训 1
- 教训 2

## Changelog
| 日期 | 变更 |
|------|------|
| YYYY-MM-DD | 初版创建 |
```
